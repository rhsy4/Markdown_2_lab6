## 1. Удирдлагын Баримт Бичгийн Зарчим

### Онолын тайлбар
**Удирдлагын баримт бичиг** гэдэг нь байгууллагын өгөгдлийн сангийг аюулгүй, найдвартай, тогтвортой ажиллуулах зорилгоор боловсруулсан **албан ёсны дүрэм, журам, заавар, стандартуудын Багц** юм.
```
Хамгаалалтын програмууд нь "цайзын хана" бол баримт бичиг нь цайзын дотоод дүрэм журам" юм. Хоёулаа зайлшгүй хэрэгтэй.
```
###  Жишээ: Баримт бичигтэй баримт бичиггүй байгууллага

```
Баримт бичиггүй байгууллага:
- Хэн дуртай нь өгөгдлийн санд хандана
- Нөөцлөлт хийгддэггүй
- Халдлага гарсан ч хариу арга хэмжээ тодорхойгүй

Баримт бичигтэй байгууллага:
- Хандалтын шатлалтай (Admin → Manager → User)
- Тогтмол нөөцлөлт хийдэг
- Нууц үгийн хатуу бодлоготой


```
### жишээ хүснэгт

| Хэрэглэгч | Унших | Засах | Устгах |
|-----------|:-----:|:-----:|:------:|
| Admin     | ✅    | ✅    | ✅     |
| Manager   | ✅    | ✅    | ❌     |
| User      | ✅    | ❌    | ❌     |


##  Нууц үг
### Онол

**Нууц үгийн бодлого** нь хэрэглэгчийн бүртгэл болон нэвтрэх мэдээллийг хамгаалахад чиглэсэн дүрэм, шаардлага, стандартуудын цогц нэгдэл юм юм.
### Сайн нууц үгийн шаардлагууд

```
Хүчтэй нууц үгийн стандарт:
Урт: Хамгийн багадаа 12 тэмдэгт
Том үсэг: A-Z  
Жижиг үсэг: a-z  
Тоо: 0-9 
усгай тэмдэгт: !@#$%^&*
Солих давтамж: 30-90 хоног тутам
```

### Жишээ: Сайн ба муу нууц үгийн харьцуулалт

```
Муу нууц үгүүд:
123456        зөвхөн тоо
password      энгийн үг
admin         хэрэглэгчийн нэр
qwerty        гарны эгнээ
1990/01/01    төрсөн огноо

Хүчтэй нууц үгийн жишээ:
P@ssw0rd#2026!    том, жижиг үсэг + тоо + тусгай тэмдэгт
Mn$ecur1ty#DB     урт, хосолсон
7h!sIsStr0ng@     санахад хялбар, хамгаалалт өндөр
```

### Нууц үгийн бодлого яагаад чухал вэ?

Ихэнх кибер халдлагууд **сул нууц үгээс** болдог:

```
Brute Force Attack     Бүх боломжит хослолыг туршина
Dictionary Attack      Нийтлэг үгсийн сангаас хайна
Credential Stuffing    Бусад сайтаас алдагдсан нууц үгийг ашиглана
```

##  Нууц үг хамгаалах

### Онолын тайлбар

Нууц үгийг өгөгдлийн санд **хэрхэн хадгалах** нь аюулгүй байдлын хамгийн чухал асуудлуудын нэг юм.

**Эрсдэл:**
- Өгөгдлийн сан алдагдвал **бүх нууц үг ил** болно
- Дотоод ажилтнууд харах боломжтой
- Хакерууд шууд нэвтэрч чадна
- Дотоод мэдээлэл алдагдах
- Эд хөрөнгөөр хохрих эрсдэлтэй байдаг

Ийм хадгалалт **ХАТУУ ХОРИГЛОГДОНО**
---
###  Encryption — Дунд Зэргийн Хамгаалалт
Нууц үгийг алгоритм болон **түлхүүр** ашиглан шифрлэж хадгалах.

```
admin123  ->  [AES Encryption + Secret Key]  ->  A$7kL!90xP==
```

**Давуу тал:** Нууц үг ойлгомжгүй болж хувирна  
**Сул тал:** Түлхүүр алдагдвал бүх нууц үг тайлагдана гэхдээ буцааж сэргээх боломжтой тул бүрэн аюулгүй биш

---

### Hashing — Орчин Үеийн Хамгийн Найдвартай Арга

Hashing нь нууц үгийг буцааж тайлах боломжгүй математик алгоритмаар хувиргадаг. Ингэснээр халдагч нараас сэргээлэх боломжтой.

```
admin123  →  [SHA-256]  →  482c811da5d5b4bc6d497fa98491638
          →  [bcrypt]   →  $2b$12$EixZaYVK1fsbw1ZfbX3OXe...
```

**Нэвтрэх үйл явц:**

```
Хэрэглэгч "admin123" оруулна
        ↓
Систем hash-лана → 482c811da5d5b4bc...
        ↓
Өгөгдлийн санд хадгалагдсан hash-тай харьцуулна
        ↓
Ижил → Нэвтрүүлнэ   |  Өөр → Татгалзана 
```
